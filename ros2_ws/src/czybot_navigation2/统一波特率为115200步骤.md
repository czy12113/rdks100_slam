# 统一波特率为 115200 的完整步骤

## 为什么要改为 115200

- **9600 太慢**：每秒最多传输约 960 字节
- **里程计帧 20 字节**：9600 波特率下最多 48Hz，控制响应有延迟
- **115200 是标准**：12 倍速度，里程计可达 576Hz，控制更流畅

---

## 修改步骤（必须按顺序执行）

### 步骤1：修改 STM32 固件波特率

在 STM32 固件的串口初始化代码中找到波特率配置：

**标准库（STM32F1/F4）：**
```c
// 在 usart.c 或 main.c 中找到类似代码
USART_InitTypeDef USART_InitStructure;
USART_InitStructure.USART_BaudRate = 9600;  // ← 找到这一行
USART_InitStructure.USART_BaudRate = 115200;  // ← 改为 115200
USART_InitStructure.USART_WordLength = USART_WordLength_8b;
USART_InitStructure.USART_StopBits = USART_StopBits_1;
USART_InitStructure.USART_Parity = USART_Parity_No;
USART_Init(USART1, &USART_InitStructure);
```

**HAL 库（STM32CubeMX）：**
```c
// 在 usart.c 中找到 MX_USARTx_UART_Init() 函数
void MX_USART1_UART_Init(void)
{
  huart1.Instance = USART1;
  huart1.Init.BaudRate = 9600;  // ← 找到这一行
  huart1.Init.BaudRate = 115200;  // ← 改为 115200
  huart1.Init.WordLength = UART_WORDLENGTH_8B;
  huart1.Init.StopBits = UART_STOPBITS_1;
  huart1.Init.Parity = UART_PARITY_NONE;
  HAL_UART_Init(&huart1);
}
```

**修改后：**
1. 重新编译固件
2. 烧录到 STM32
3. 重启 STM32

---

### 步骤2：修改 TJC 串口屏波特率

在 TJC HMI 软件中：

1. 打开你的 HMI 项目文件（.HMI）
2. 点击菜单栏 **"程序"** → **"程序属性"**
3. 在弹出窗口中找到 **"串口设置"** 或 **"Serial Settings"** 标签页
4. 找到 **"波特率"** 下拉框，选择 **115200**
5. 点击 **"确定"** 保存
6. 点击 **"编译"** 按钮（或按 F5）
7. 编译成功后，点击 **"上传"** 按钮烧录到串口屏
8. **拔掉串口屏电源，等待 3 秒，重新上电**（必须重启才能生效）

### 步骤3：验证硬件波特率是否修改成功

在 RDK S100 上运行波特率扫描脚本：

```bash
cd ~/ros2_ws/src/czybot_navigation2/scripts
bash scan_baudrate.sh
```

**期望输出：**
```
── 扫描 /dev/ttyUSB0 ──────────────────────────────
  ✓ 115200 bps: 收到非零数据！
    HEX: bb6601...
    ✓✓ 检测到里程计帧头 BB 66！这是STM32！
        STM32波特率 = 115200

── 扫描 /dev/ttyUSB1 ──────────────────────────────
  ✓ 115200 bps: 收到非零数据！
    HEX: 650c01ffffff...
```

**如果 115200 那一行仍然是"无数据"或"全零"，说明硬件波特率没有修改成功，需要重新检查步骤1和步骤2。**

## 常见问题

### Q1：STM32 固件源码在哪里？
A：通常在你的开发电脑上，不在 RDK S100 上。需要用 Keil/IAR/STM32CubeIDE 打开。

### Q2：我不知道 STM32 固件是谁写的
A：联系提供小车的厂商或开发者，获取固件源码。

### Q3：能不能只改 ROS 代码，不改硬件？
A：不能。硬件和软件波特率必须一致，否则无法通信。

### Q4：改完后小车又不动了怎么办？
A：运行 `bash scan_baudrate.sh` 确认硬件波特率，然后运行 `bash diagnose_stm32.sh` 诊断。
