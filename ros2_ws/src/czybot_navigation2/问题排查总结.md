# TJC串口屏控制小车问题排查总结

## 问题概述
TJC串口屏按钮无法控制小车运动，手动发布 `/cmd_vel` 也无效。

---

## 根本原因（双重故障）

### 故障1：TJC串口屏配置错误 ⚠️
**症状：** 串口监听只收到 `00 00 00 FF FF FF`，无触摸事件  
**原因：** `printh` 命令未正确配置在按钮事件中  
**影响：** TJC → ROS 链路断开

### 故障2：STM32通信问题 ⚠️
**症状：** 手动 `ros2 topic pub /cmd_vel` 也无法让小车动  
**原因：** 待诊断（串口协议/接线/固件）  
**影响：** ROS → STM32 链路断开

---

## 完整控制链路

```
┌─────────────┐   串口    ┌──────────────────┐   ROS话题   ┌──────────────────┐   串口    ┌────────┐   PWM    ┌──────┐
│  TJC串口屏  │ ────────> │ tjc_hmi_bridge   │ ─────────> │  stm32_bridge    │ ────────> │ STM32  │ ──────> │ 电机 │
│  (按钮)     │ ttyUSB1   │  (ROS节点)       │  /cmd_vel  │   (ROS节点)      │ ttyUSB0   │        │         │      │
└─────────────┘ 115200    └──────────────────┘            └──────────────────┘ 115200    └────────┘         └──────┘
     ❌ 故障1                                                      ❌ 故障2
```

## 修复方案

### 🔧 修复故障1：TJC串口屏配置

#### 正确的HMI配置步骤
1. **设置全局波特率为115200**
   - 程序 → 程序属性 → 串口设置 → 115200

2. **配置按钮事件**（以前进按钮为例）
   - 双击按钮组件
   - 选择 **"弹起事件"** 标签页
   - 输入：`printh 65 00 0C 01 FF FF FF`
   - 点击确定

3. **所有按钮配置**
   ```
   前进(ID=12): printh 65 00 0C 01 FF FF FF
   后退(ID=13): printh 65 00 0D 01 FF FF FF
   急停(ID=14): printh 65 00 0E 01 FF FF FF
   左转(ID=15): printh 65 00 0F 01 FF FF FF
   停止(ID=16): printh 65 00 10 01 FF FF FF
   右转(ID=17): printh 65 00 11 01 FF FF FF
   ```

4. **编译、烧录、重启**
   - 编译（F5）
   - 上传到串口屏
   - **重新上电串口屏**（必须！）

#### 验证方法
```bash
# 在RDK S100上执行
cd ~/ros2_ws/src/czybot_navigation2/scripts
bash diagnose_tjc.sh
```

**期望结果：** 按按钮时收到 `65 00 0C 01 FF FF FF`

#### 可能的问题和解决方法

**问题A：串口设备插反了**
- 症状：诊断脚本显示 ttyUSB0 无响应
- 解决：交换 ttyUSB0 和 ttyUSB1 的连接
- 修改配置文件中的端口参数

**问题B：STM32未上电或固件未运行**
- 症状：发送命令无响应，无里程计数据
- 解决：检查STM32电源，重新烧录固件

**问题C：波特率不匹配**
- 症状：收到乱码数据
- 解决：确认STM32固件波特率为115200

**问题D：协议格式不匹配**
- 症状：STM32收到数据但不执行
- 解决：检查STM32固件的协议解析代码
- 参考：[`stm32_bridge.py:111`](scripts/stm32_bridge.py:111) 中的帧格式

### 串口协议格式

**TJC → ROS（触摸事件）**
```
格式: 0x65 [页面ID] [组件ID] [事件类型] 0xFF 0xFF 0xFF
示例: 65 00 0C 01 FF FF FF  (前进按钮按下)
```

**ROS → STM32（速度控制）**
```
格式: 0xAA 0x55 0x01 [linear_int16_LE] [angular_int16_LE] 0x00 [checksum] 0x0D
示例: AA 55 01 C8 00 00 00 C9 0D  (前进200mm/s)
```

**STM32 → ROS（里程计）**
```
格式: 0xBB 0x66 0x01 [pos_x 4B] [pos_y 4B] [yaw 2B] [vx 2B] [vth 2B] [reserved 2B] [checksum] 0x0D
长度: 20字节
```

## 常见错误

### ❌ 错误1：TJC使用 `print` 而不是 `printh`
```
错误: print 65 00 0c 01 ff ff ff
正确: printh 65 00 0C 01 FF FF FF
```

### ❌ 错误2：命令放在错误位置
```
错误: 放在"初始化事件"或其他位置
正确: 必须放在"弹起事件"或"按下事件"
```

### ❌ 错误3：配置后未重启串口屏
```
解决: 重新上电串口屏，配置才会生效
```

### ❌ 错误4：串口被占用
```bash
# 检查占用
lsof /dev/ttyUSB1

# 停止占用进程
ros2 node kill /tjc_hmi_bridge
```
